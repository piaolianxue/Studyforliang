嵌入式系统驱动架构设计与分层优化

    APP层           Devive层        BSP层
    执行业务逻辑    设备命令解析     时序控制和硬件驱动

分层解耦的好处
   更换芯片平台时，APP层无需修改 
   设备层保持稳定，只需适配新I2C驱动
   BSP层实现新硬件驱动接口即可
   大幅降低移植成本和风险

OLED IIC运行机制
    主机设备开始初始化数据通讯。开始条件通过将SDA从高拉低而SCL保持高建立
    SCL拉高期间，SDA电平保持不变，SCL拉低，SDA开始变换电平
    当SCL为高电平时，SDA由低变高，IIC结束
写操作
    主机发送开始信号-> 7位设备地址 + 1位Control_Byte（0） -> Ack（从机）  -> 8位数据 -> Ack（从机） -> 主机发送结束信号
读操作
    主机发送开始信号-> 7位设备地址 + 1位Control_Byte（1） -> Ack（从机）  -> 8位数据（从机） -> Ack -> 主机发送结束信号

    前8位发送数据 第9位应答

从机让主机等待
    从机需要将SCL拉低
    原因：  对于iic总线上的设备来说，只要有一个人拉低，总线就是低。
            当主机数据传输完之后哪怕把SCL拉高，准备下一组数据传输，从机拉低SCL就会使得SCL始终为低，主机就会知道，从机没有就位。
